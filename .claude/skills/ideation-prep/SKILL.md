# 企画仕込みシート作成

お題を受けてから企画会議・ブレストまでの「下ごしらえ」を代行する。
ゴールは**アイデアを出すことではなく、人間が良いアイデアを出せる状態を最短で作ること**。

## 入力（不足分は推測で埋めて仮定を明記）

- お題 / クライアント / 商材
- ターゲット・伝えたいこと（わかれば）
- 制約: 予算感・媒体・時期・NGなど（わかれば）

## 手順

### 1. お題の分解（課題の再定義候補）

表面のお題の裏にある課題を 3〜5 通りに言い換える。
「本当の敵は何か」（無関心？誤解？競合？習慣？）を明示する。

### 2. 社内ナレッジを引く（必須）

```bash
node scripts/search-cases.mjs <お題キーワード…> --limit 15
node scripts/search-cases.mjs --tag <関連タグ> --year 2023-2026 --limit 10
```

- ヒットした事例は検証済み。切り口の種・参考事例・既視感チェックの3用途に使う
- タグ語彙: `data/tag-vocabulary.json`。関連しそうな Tech/Form/Theme を2〜3軸試す

### 3. トレンド速調べ（軽く）

WebSearch でお題領域の直近トレンドを2〜3クエリだけ確認（深掘りは `/case-research` に回す）。
外部の参考事例を数集める必要が出たら `case-collector` に委任し、受賞・数値を書く事例だけ `award-verifier` に通す。
**シートの執筆（課題再定義・切り口マップ）はメインが行う**（発想の質が本体のため委任しない）。

### 4. 仕込みシート出力

保存先: `ideation/<お題slug>-prep.md`（`ideation/` は無ければ作成）

```markdown
# 企画仕込みシート: <お題>
作成: <日付> / 前提: <仮定した事項>

## 1. お題の分解
- そもそもの課題（再定義候補 3〜5）
- 本当の敵: <無関心/誤解/競合/…>

## 2. 切り口マップ（5〜7方向）
| # | 切り口 | 一言コンセプト | 参考事例（社内DB/外部） | 強み | リスク |

## 3. 参考事例ショートリスト（検証済み）
（search-cases.mjs の結果から、切り口に対応づけて 5〜10件）

## 4. 既視感アラート
（お題に近すぎる有名事例。「これと同じと言われる」ものを先に潰す）

## 5. トレンドメモ
（直近の空気。2〜3項目）

## 6. ブレストへの問い
（会議で議論すべき問いを5個。Yes/Noで答えられない形で）
```

## してはいけないこと

- 完成した企画案を大量に並べない（判断材料を作るのが仕事。アイデアの断片は切り口マップ内の一言に留める）
- 検索でヒットしなかったからと事例をでっち上げない。外部事例で受賞・数値を書くときは出典URLを付ける
