# リサーチ→HTMLデッキ変換

Markdown のリサーチレポートを、1ファイルで完結するプレゼン用 HTML デッキに変換する。
PowerPoint を生成するのではなく「ブラウザで開けばそのまま発表できる／Cmd+P でPDFになる」形を正とする。

**実行体制**: 変換の実作業は `deck-builder` エージェントに委任する（元MDのパスと以下の手順・規約を渡す）。メインは元ネタの選定と完成デッキの最終確認のみ。

## 手順

1. **元ネタを読む**: 指定されたMD（`*_RESEARCH.md` / `ideation/*.md`）。指定がなければ直近更新のレポートを候補として確認する
2. **テンプレを土台にする**: このスキルの `template.html` を読み、その構造・CSS・JSを**そのまま**使う（`{{...}}` を実データで置換し、切り口スライドを章の数だけ複製する）
3. **情報を「間引く」**（ここが本体）:
   - 1スライド = 1メッセージ。レポートの全文を貼らない
   - 表紙 → サマリー（結論3行）→ 切り口ごとに1枚（代表事例2〜4件、コア体験を2行まで）→ Takeaway（お題への応用）
   - 事例は元レポートで**検証済みのものだけ**使う。`⚠️未検証` の事例はデッキに入れない
   - 受賞情報は正確なレベル・部門のまま短縮する（Gold を受賞とだけ書くのはOK、Shortlist を受賞と書くのはNG）
4. **保存**: `decks/<元ファイルslug>.html`（`decks/` は無ければ作成）
5. **確認**: `open decks/<...>.html` でブラウザを開いて渡す

## デザイン規約（template.html に準拠）

- ダーク背景 + アクセント1色。色を増やさない
- フォントサイズはテンプレ既定を維持（会議室の後ろから読める前提。文字を小さくして詰め込まない）
- 事例カードは1スライド最大4枚。溢れたらスライドを分ける
- 各スライド下部に出典URLを `.src` で残す（口頭で聞かれたとき用）

## してはいけないこと

- 外部CDN・外部画像を参照しない（オフラインの会議室でも開ける自己完結ファイルにする）
- 元レポートにない事実・数値を足さない。要約で受賞レベルを変えない
- クライアント名入りデッキを社外共有可能な場所に置かない（`decks/` はローカルのみ。gitignore 推奨）
